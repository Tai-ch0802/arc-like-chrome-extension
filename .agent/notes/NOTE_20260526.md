## 技術決策
- **ESM/CJS 取捨**：Jest 30 + esbuild transform 比加 babel-jest 兩個 devDep 乾淨；用 .mjs 副檔名區分「單元測試（ESM）」vs「Puppeteer 測試（CJS）」風格。
- **race 修法不一刀切 await**：保留作者刻意的啟動性能優化，只在「cold start」場景 await。warm start 透過事件補償尚未完成的 cache rebuild。
- **重構不過度設計**：抽 splitView + listeners 兩個子模組是「真痛點解套」（splitView 邏輯密集、listeners 5 個 handler 行數大）；其他單函式 renderer 硬拆會增加 module 切換成本而無收益。
