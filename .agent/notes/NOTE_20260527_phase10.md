## 技術決策

- **保留 Arc-like 品牌 vs. 完全重命名**：選保留 + 加副標。Arc browser 是 high-volume search keyword，完全重命名會丟失既有 SEO 與既有使用者品牌認知。
- **14 語 mass apply 用 Python script**：手動 14 × 3 = 42 個檔案 edit 太慢且易出錯。Python script + AST parse 載入 DATA dict 一次 batch apply，可重現可 audit。第二版用 AST literal_eval 避開 importlib 雙跑問題。
- **「H1 + intro」14 語完整、「features 詳細列表」對 EN+zh_TW 完整 / 其他 12 語精簡**：詳細 features 翻譯需 native speaker quality，v1 先在 EN+zh_TW 完整呈現，其他 12 語 store_description 內已加 9-bullet KW section 已足夠暴露差異化。
- **Landing page 純英文**：12 語版本是 SEO 工作，第一版聚焦英文，跨語 landing 留 v2。
- **不寫 promotion 截圖**：截圖需設計工具 + 實機畫面，user 自製。我已在 PR body 列規格建議。
- **不改 manifest.json `description` 欄位**：用 `__MSG_extensionDescription__` 引用 i18n，已透過 14 個 messages.json 更新生效。
- **保留既有 store_description 內容**：只 H1 + 加新 section / callout，避免破壞既有 SEO 內容與評分連結。
